# Architecture Decision Records

This directory records architectural decisions that impose long-lived constraints on Gryd Lock's browser-extension design.

ADRs explain **why** a decision exists, the alternatives considered, and the consequences future contributors must preserve or intentionally supersede. They complement the README's description of the current system; they do not replace code, tests, or security documentation.

## When to add an ADR

Create an ADR for changes to shared abstractions, execution-context responsibilities, wallet interception contracts, cross-context messaging, graceful-degradation behavior, security boundaries, persistence, or build/runtime strategy.

See [CONTRIBUTING.md](../../CONTRIBUTING.md#architecture-decision-records) for the complete process and examples.

## Status values

- **Proposed** — under discussion and not yet binding.
- **Accepted** — approved and expected to guide implementation.
- **Deprecated** — retained for history but no longer recommended.
- **Superseded by ADR-NNNN** — replaced by a newer decision.
- **Rejected** — considered but not adopted.
